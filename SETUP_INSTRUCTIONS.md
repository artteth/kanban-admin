# 📋 Kanban Board v2 - Инструкция по установке и настройке

## 🎯 Что нового в версии 2

1. **Система пользователей** - каждый пользователь видит только свои задачи
2. **Назначение задач** - при создании задачи выбирается исполнитель
3. **Админская панель** - полный контроль над всеми задачами и пользователями
4. **CRUD операций** - создание, просмотр, редактирование, удаление задач и пользователей
5. **Фильтрация** - задачи можно фильтровать по пользователю, статусу и поиску

## ⚙️ Пошаговая настройка

### Шаг 1: Создание Google Таблицы

1. Откройте https://sheets.google.com
2. Создайте новую таблицу
3. Назовите её "Kanban Tasks v2"

### Шаг 2: Настройка листов

#### Лист "задания" (существующий или новый)

Добавьте колонку **H** с заголовком `AssignedTo`:

| A | B | C | D | E | F | G | H |
|---|---|---|---|---|---|---|---|
| **ID** | **Задание** | **Статус** | **Дата начала** | **Дата конца** | **Время выполнения** | **Плановая дата** | **AssignedTo** |
| task_123 | Купить молоко | todo | 2026-03-05 | | | 2026-03-06 | user_1 |

#### Лист "Пользователи" (новый)

Создайте новый лист с именем `Пользователи` и заголовками:

| A | B | C | D |
|---|---|---|---|
| **UserID** | **Имя** | **TelegramID** | **Роль** |
| user_1 | Артём | @artem | user |
| user_2 | Игорь | @igor | user |
| admin_1 | Админ | @admin | admin |

### Шаг 3: Получение ID таблицы

Скопируйте ID таблицы из URL:

```
https://docs.google.com/spreadsheets/d/1TsUjce91h44W_PF4dzCqCwGTB_jqhjJxRWBsLiGPmjE/edit
                                            ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                                            ЭТО ВАШ ID ТАБЛИЦЫ
```

### Шаг 4: Настройка Google Apps Script

1. В таблице: **Расширения** → **Apps Script**
2. Удалите старый код (если есть)
3. Скопируйте содержимое файла `cod_v2.gs`
4. Вставьте в редактор Apps Script

#### Обновите константы в начале файла:

```javascript
const SPREADSHEET_ID = 'ВАШ_ID_ТАБЛИЦЫ';  // Вставьте ID из шага 3
const SHEET_NAME_TASKS = 'задания';        // Имя листа с задачами
const SHEET_NAME_USERS = 'Пользователи';   // Имя листа пользователей
const TELEGRAM_BOT_TOKEN = 'ВАШ_ТОКЕН';    // Токен бота (если нужен)
```

### Шаг 5: Развёртывание веб-приложения

1. В редакторе Apps Script нажмите **Развернуть** → **Новое развёртывание**
2. Выберите тип: **Веб-приложение**
3. Настройте:
   - **Описание**: Kanban v2
   - **Запуск от имени**: Меня (ваш email)
   - **Кто имеет доступ**: Все пользователи (или "Все, у кого есть аккаунт Google")
4. Нажмите **Развернуть**
5. Разрешите доступ при запросе
6. **Скопируйте URL веб-приложения** (выглядит как `https://script.google.com/macros/s/.../exec`)

### Шаг 6: Настройка frontend файлов

Откройте каждый файл и замените `API_URL`:

#### В файлах:
- `select_user.html` (строка ~630)
- `index_v2.html` (строка ~870)
- `admin.html` (строка ~630)

#### Замените:
```javascript
const API_URL = 'https://script.google.com/macros/s/ВАШ_URL/exec';
```

На ваш URL из шага 5.

### Шаг 7: Добавление первых пользователей

1. Откройте `select_user.html` в браузере
2. Нажмите "➕ Добавить пользователя"
3. Создайте первого пользователя (например, себя)
4. Создайте админа (роль "admin")

### Шаг 8: Проверка работы

1. Откройте `select_user.html`
2. Выберите пользователя
3. Нажмите "Продолжить"
4. Создайте задачу с назначением на пользователя
5. Проверьте, что задача видна только этому пользователю
6. Откройте `admin.html` - должны быть видны все задачи

## 🎨 Как использовать

### Обычный пользователь

1. Откройте `select_user.html`
2. Выберите своё имя
3. Нажмите "Продолжить" → откроется Kanban-доска
4. Для создания задачи:
   - Нажмите "+ Добавить задачу"
   - Введите описание
   - Выберите исполнителя (себя или другого)
   - Укажите плановую дату (опционально)
   - Нажмите "Добавить"
5. Перемещайте задачи между колонками:
   - Кнопками "Вперёд" / "Назад"
   - Drag-and-drop между колонками

### Администратор

#### Через админскую панель (`admin.html`):

1. **Просмотр всех задач**
   - Все задачи отображаются в таблице
   - Фильтры: по пользователю, статусу, поиску

2. **Создание задачи**
   - Нажмите "➕ Добавить задачу"
   - Заполните поля
   - Выберите исполнителя
   - Сохраните

3. **Редактирование задачи**
   - Нажмите ✏️ в строке задачи
   - Измените поля
   - Сохраните

4. **Удаление задачи**
   - Нажмите 🗑️ в строке задачи
   - Подтвердите удаление

5. **Управление пользователями**
   - Секция "Пользователи" внизу страницы
   - Добавление, редактирование, удаление
   - Видно количество задач у каждого

#### Через обычную доску (`index_v2.html`):

Если админ заходит через обычную доску:
- Видит все задачи (как админ)
- Может создавать задачи с любым исполнителем

## 🔐 Роли и права

| Роль | Права |
|------|-------|
| **admin** | Видит все задачи, CRUD всех задач, управление пользователями |
| **user** | Видит только свои задачи, создание задач, редактирование через админа |

## 🛠️ API Endpoints

### GET запросы

```
?mode=jsonp&action=getTasks&userId=USER_ID
?mode=jsonp&action=getUsers&callback=myCallback
```

### POST запросы (JSON)

```json
// Получить задачи
{ "action": "getTasks", "userId": "user_1" }

// Получить пользователей
{ "action": "getUsers" }

// Добавить задачу
{
  "action": "addTask",
  "title": "Текст задачи",
  "plannedDate": "2026-03-06",
  "assignedTo": "user_1"
}

// Обновить задачу
{
  "action": "updateTask",
  "id": "task_123",
  "title": "Новый текст",
  "status": "in-progress",
  "plannedDate": "2026-03-07",
  "assignedTo": "user_2"
}

// Удалить задачу
{ "action": "deleteTask", "id": "task_123" }

// Добавить пользователя
{
  "action": "addUser",
  "name": "Имя",
  "telegramId": "@telegram",
  "role": "user"
}

// Обновить пользователя
{
  "action": "updateUser",
  "userId": "user_1",
  "name": "Новое имя",
  "telegramId": "@newtelegram",
  "role": "admin"
}

// Удалить пользователя
{ "action": "deleteUser", "userId": "user_1" }
```

## 🐛 Возможные проблемы и решения

### Задачи не загружаются

**Проблема:** Ошибка CORS или "JSONP request failed"

**Решение:**
1. Проверьте, что `API_URL` правильный
2. Убедитесь, что веб-приложение развёрнуто с доступом "Все пользователи"
3. Проверьте консоль браузера на ошибки

### Пользователи не сохраняются

**Проблема:** Ошибка при сохранении пользователя

**Решение:**
1. Проверьте, что лист "Пользователи" существует
2. Проверьте заголовки столбцов (UserID, Имя, TelegramID, Роль)
3. Убедитесь, что `SPREADSHEET_ID` правильный

### Задачи видны всем вместо конкретного пользователя

**Проблема:** Не работает фильтрация

**Решение:**
1. Проверьте, что колонка `AssignedTo` существует в листе "задания"
2. Убедитесь, что `assignedTo` сохраняется при создании задачи
3. Проверьте, что `userId` передаётся в API

### Админ не видит все задачи

**Проблема:** Админ видит только некоторые задачи

**Решение:**
1. В `admin.html` проверьте, что `userId: 'admin'` передаётся в API
2. В `cod_v2.gs` проверьте функцию `getTasksData()` - должна возвращать все задачи для 'admin'

## 📊 Миграция со старой версии

Если у вас уже есть задачи в таблице:

1. **Добавьте колонку AssignedTo** в лист "задания"
2. **Заполните старые задачи:**
   - Вариант A: Назначьте все старые задачи на одного пользователя (например, admin)
   - Вариант B: Оставьте пустыми - они будут видны всем (до первой модификации)
3. **Создайте лист "Пользователи"** с текущими пользователями
4. **Обновите backend** на `cod_v2.gs`
