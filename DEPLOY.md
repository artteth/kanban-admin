# 🚀 Инструкция по деплою Kanban Board v2

## 📋 Что нужно для запуска онлайн

### 1️⃣ Google Таблица (База данных)

**Создайте новую таблицу:**

1. Откройте https://sheets.google.com
2. Нажмите **"+ Пустой файл"**
3. Назовите таблицу: `Kanban Tasks v2`

**Настройте листы:**

#### Лист 1: "задания"
1. Переименуйте первый лист в `задания`
2. Создайте заголовки в первой строке:

| A | B | C | D | E | F | G | H |
|---|---|---|---|---|---|---|---|
| **ID** | **Задание** | **Статус** | **Дата начала** | **Дата конца** | **Время выполнения** | **Плановая дата** | **AssignedTo** |

3. **Скопируйте ID таблицы** из URL:
   ```
   https://docs.google.com/spreadsheets/d/1ABC123xyz456_ABC123xyz456/edit
                                               ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                                               ЭТО ВАШ ID ТАБЛИЦЫ (скопируйте)
   ```

#### Лист 2: "Пользователи"
1. Нажмите **"+"** (добавить лист)
2. Переименуйте в `Пользователи`
3. Создайте заголовки:

| A | B | C | D |
|---|---|---|---|
| **UserID** | **Имя** | **TelegramID** | **Роль** |

4. Добавьте первого пользователя (админа):

| A | B | C | D |
|---|---|---|---|
| admin_1 | Админ | @admin | admin |

---

### 2️⃣ Google Apps Script (Backend)

**Настройте скрипт:**

1. В таблице: **Расширения** → **Apps Script**

2. **Откройте файл `v2/cod_v2.gs`** из проекта и скопируйте всё содержимое

3. **Вставьте в редактор Apps Script** (удалите старый код если есть)

4. **Обновите константы** в начале файла (строки 4-8):
   ```javascript
   const SPREADSHEET_ID = '1ABC123xyz456_ABC123xyz456';  // ← Ваш ID из шага 1
   const SHEET_NAME_TASKS = 'задания';
   const SHEET_NAME_USERS = 'Пользователи';
   const TELEGRAM_BOT_TOKEN = '8664566561:AAEV11uRMZIxmqjcoQybafCWAmQhdoQdbXs';
   ```

5. **Сохраните проект:**
   - Нажмите 💾 или Ctrl+S (Cmd+S)
   - Назовите проект: `Kanban v2 Backend`

**Разверните веб-приложение:**

1. Нажмите **Развернуть** → **Новое развёртывание**

2. Настройте:
   - **Тип**: Веб-приложение
   - **Описание**: `Kanban v2`
   - **Запуск от имени**: Меня (ваш email)
   - **Кто имеет доступ**: **Все пользователи** ⚠️

3. Нажмите **Развернуть**

4. **Разрешите доступ:**
   - Нажмите "Проверить разрешения"
   - Выберите ваш Google аккаунт
   - "Дополнительные настройки" → "Перейти к ... (небезопасно)"
   - "Разрешить"

5. **Скопируйте URL веб-приложения:**
   ```
   https://script.google.com/macros/s/AKfycbwXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX/exec
   ```
   **Этот URL понадобится для frontend!**

### 3️⃣ Frontend (Хостинг)

Есть 2 варианта:

#### Вариант A: GitHub Pages (рекомендуется) ⭐

**1. Подготовьте файлы:**

Скопируйте эти файлы из `v2/` в отдельную папку:
```
deploy/
├── index.html          (копия v2/select_user_local.html)
├── admin.html          (копия v2/admin_local.html)
├── dashboard.html      (копия v2/index_local.html)
└── README.md
```

**2. Переименуйте:**
- `select_user_local.html` → `index.html` (главная страница)
- `admin_local.html` → `admin.html`
- `index_local.html` → `dashboard.html`

**3. Обновите API_URL в каждом файле:**

Найдите строку (примерно строка 820 в index.html):
```javascript
const API_URL = 'https://script.google.com/macros/s/.../exec';
```

Замените на **ваш URL** из шага 2!

**4. Загрузите на GitHub:**

```bash
# В папке с файлами deploy/
git init
git add .
git commit -m "Kanban v2 frontend"
git branch -M main
git remote add origin https://github.com/ВАШ_НИК/kanban-v2.git
git push -u origin main
```

**5. Включите GitHub Pages:**

1. Откройте репозиторий на GitHub
2. **Settings** → **Pages**
3. **Source**: Deploy from a branch
4. **Branch**: main / root
5. **Save**

Через 1-2 минуты сайт будет доступен:
```
https://ВАШ_НИК.github.io/kanban-v2/
```

#### Вариант B: Любой хостинг

## 🐛 Решение проблем

### Ошибка: "JSONP request failed"

**Причина:** Неправильный API_URL или нет доступа

**Решение:**
1. Проверьте, что URL скопирован полностью
2. Проверьте права доступа: **Развернуть** → **Управление развёртываниями** → **Все пользователи**
3. Откройте консоль браузера (F12) → посмотрите детали ошибки

### Ошибка: "Пользователи не загружаются"

**Причина:** Лист "Пользователи" не создан или неправильно назван

**Решение:**
1. Проверьте имя листа: должно быть точно `Пользователи`
2. Проверьте заголовки: `UserID`, `Имя`, `TelegramID`, `Роль`
3. Добавьте хотя бы одного пользователя вручную

### Ошибка: "Задачи не сохраняются"

**Причина:** Нет колонки `AssignedTo`

**Решение:**
1. Откройте лист "задания"
2. Добавьте колонку **H** с заголовком `AssignedTo`
3. Сохраните таблицу

### Задачи видны всем вместо конкретного пользователя

**Причина:** Не передаётся `userId` в API

**Решение:**
1. Проверьте, что `API_URL` правильный во всех файлах
2. Проверьте консоль браузера на ошибки
