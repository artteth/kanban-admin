# 🧪 Тестирование Kanban Board v2

## Вариант 1: Быстрый тест UI (без backend)

### Шаг 1: Откройте файлы локально

Просто откройте HTML файлы в браузере:

```bash
# В терминале, находясь в папке проекта:
open v2/select_user.html      # macOS
xdg-open v2/select_user.html  # Linux
start v2/select_user.html     # Windows
```

**Что вы увидите:**
- ✅ Страница выбора пользователя загрузится
- ❌ При попытке загрузить/сохранить данные будет ошибка (нет backend)

Это нормально — UI работает, но API ещё не настроен.

## Вариант 2: Полноценный тест с Google Apps Script

### 📋 Шаг 1: Подготовка Google Таблицы

1. **Создайте новую таблицу** (или используйте существующую):
   - https://sheets.google.com → Создать
   
2. **Проверьте лист "задания"**:
   - Должен быть лист с именем `задания`
   - Добавьте колонку **H** с заголовком `AssignedTo` (если нет)
   
   ```
   | A | B | C | D | E | F | G | H |
   |---|---|---|---|---|---|---|---|
   | ID | Задание | Статус | Дата начала | Дата конца | Время выполнения | Плановая дата | AssignedTo |
   ```

3. **Создайте лист "Пользователи"**:
   - В таблице: "+" (добавить лист)
   - Переименуйте в `Пользователи`
   - Добавьте заголовки:
   
   ```
   | A | B | C | D |
   |---|---|---|---|
   | UserID | Имя | TelegramID | Роль |
   ```

4. **Скопируйте ID таблицы** из URL:
   ```
   https://docs.google.com/spreadsheets/d/1TsUjce91h44W_PF4dzCqCwGTB_jqhjJxRWBsLiGPmjE/edit
                                               ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
   ```

### Шаг 2: Настройка Google Apps Script

1. **Откройте редактор скриптов**:
   - В таблице: **Расширения** → **Apps Script**

2. **Вставьте код**:
   - Скопируйте содержимое `v2/cod_v2.gs`
   - Вставьте в редактор (удалив старый код)

3. **Обновите константы** (в начале файла):
   ```javascript
   const SPREADSHEET_ID = 'ВАШ_ID_ТАБЛИЦЫ';  // Вставьте из шага 1.4
   const SHEET_NAME_TASKS = 'задания';
   const SHEET_NAME_USERS = 'Пользователи';
   const TELEGRAM_BOT_TOKEN = '8664566561:AAEV11uRMZIxmqjcoQybafCWAmQhdoQdbXs';
   ```

4. **Сохраните** (Ctrl+S / Cmd+S)

### Шаг 3: Развёртывание

1. **Развернуть как веб-приложение**:
   - Нажмите **Развернуть** → **Новое развёртывание**
   - Тип: **Веб-приложение**
   - Описание: `Kanban v2`
   - Запуск от: **Меня**
   - Доступ: **Все пользователи** (или "Все, у кого есть аккаунт Google")

2. **Нажмите "Развернуть"**

3. **Разрешите доступ**:
   - Нажмите "Проверить разрешения"
   - Выберите ваш Google аккаунт
   - "Дополнительные настройки" → "Перейти к ... (небезопасно)"
   - "Разрешить"

4. **Скопируйте URL** веб-приложения:
   ```
   https://script.google.com/macros/s/AKfycbw.../exec
   ```

### Шаг 4: Настройка frontend

1. **Откройте файлы для редактирования**:
   - `v2/select_user.html` (строка ~630)
   - `v2/index_v2.html` (строка ~870)
   - `v2/admin.html` (строка ~630)

2. **Замените API_URL**:

## 🐛 Отладка проблем

### Ошибка: "JSONP request failed"

**Причина:** Неправильный API_URL или нет доступа

**Решение:**
1. Проверьте, что URL скопирован полностью
2. Проверьте права доступа: **Развернуть** → **Управление развёртываниями** → **Все пользователи**
3. Откройте консоль браузера (F12) → посмотрите детали ошибки

### Ошибка: "Пользователь не найден"

**Причина:** Лист "Пользователи" пуст или неправильно назван

**Решение:**
1. Проверьте имя листа: должно быть точно `Пользователи`
2. Проверьте заголовки: `UserID`, `Имя`, `TelegramID`, `Роль`
3. Добавьте хотя бы одного пользователя вручную в таблицу

### Задачи не сохраняются

**Причина:** Нет колонки `AssignedTo`

**Решение:**
1. Откройте лист "задания"
2. Добавьте колонку H с заголовком `AssignedTo`
3. Для старых задач заполните `AssignedTo` (или оставьте пустым)

### Задачи видны всем вместо конкретного пользователя

**Причина:** Не передаётся `userId` в API

**Решение:**
1. Проверьте в `index_v2.html`, что `currentUserId` загружается из URL или localStorage
2. Проверьте в `cod_v2.gs`, что `getTasksData()` фильтрует по userId

## 📞 Если что-то пошло не так

1. **Проверьте логи Apps Script**:
   - В редакторе скриптов: **ExpLog** → **Просмотр журналов**

2. **Проверьте консоль браузера**:
   - F12 → Console → ищите ошибки

3. **Проверьте Google Таблицу**:
   - Данные действительно записываются?
   - Правильные ли значения в колонках?

4. **Перезагрузите страницу** с очисткой кэша:
   - Cmd+Shift+R (macOS)
   - Ctrl+Shift+R (Windows/Linux)
